주요 기능

회원가입 & 로그인: 카카오 로그인 API





카카오 API Access_token을 이용하여 사용자 정보를 조회

Access Token은 사용자가 로그인 인증에 성공했을 시 부여되는 수명이 하루가 되지 않는 토큰이기 때문에 grant_type, client_id, redirect_uri, code 파라미터의 값들을 POST로 요청해 API를 호출할 때 사용하는 access_token과 해당 토큰의 만료시간, 토큰을 갱신할 수 있는 refresh_token을 받았습니다.



회원가입 후 추가 정보 입력







카카오 로그인 API에서 권한 부족으로 이름과 전화번호를 받아올 수 없어서 회원가입 직후 다음 페이지로 이동하여 이름, 전화번호, 주소를 사용자가 직접 입력하고 Form 태그의 POST방식을 이용하여 입력한 데이터 값을 DB에 저장하도록 구현했습니다.



QnA 게시판

- 게시글 조회



Thymeleaf th:each 반복문을 사용해 DB에 저장되어있는 게시글 목록을 조회하고 작성일순으로 출력하도록 구현했습니다.


- 게시글 작성





입력된 값을 POST로 받아오고 글 작성자만 수정, 삭제가 가능하도록 session을 이용해 글쓴이의 ID 정보를 받아와서 ID 값과 입력값을 DB에 저장하도록 구현했습니다.


- 게시글 수정, 삭제



게시글 작성자만 수정, 삭제가 가능하게 String으로 로그인한 사용자의 seq와 게시글 작성자의 seq가 일치할 때만 수정, 삭제 버튼이 활성화 되도록 구현했습니다.

수정을 완료한 경우 게시글 상세 페이지로 이동, 삭제를 완료한 경우 게시글 목록으로 이동하게 했습니다.


- 게시글 상세 조회 & 댓글 조회, 등록, 삭제



댓글 작성자만 삭제 할 수 있게 String으로 로그인한 사용자의 seq와 댓글 작성자의 seq가 일치할 때만 삭제 버튼이 활성화 되도록 구현했습니다.
